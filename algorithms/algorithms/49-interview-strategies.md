# Interview Strategies - Complete Guide

## Overview
Proven strategies for acing technical interviews at Senior/Staff/Principal level.

---

## Before the Interview

### Preparation Timeline
- **3 months out**: Start with Easy problems, review fundamentals
- **2 months out**: Focus on Medium problems, learn patterns
- **1 month out**: Practice Hard problems, mock interviews
- **1 week out**: Review notes, practice communication

### Study Plan
1. **Week 1-2**: Arrays, strings, hash tables
2. **Week 3-4**: Linked lists, stacks, queues
3. **Week 5-6**: Trees, graphs
4. **Week 7-8**: DP, backtracking
5. **Week 9-10**: Advanced topics, system design
6. **Week 11-12**: Mock interviews, review

---

## During the Interview

### Problem-Solving Framework
1. **Clarify** (2-3 min): Ask questions, understand constraints
2. **Examples** (2-3 min): Work through examples, edge cases
3. **Approach** (3-5 min): Discuss brute force, then optimize
4. **Code** (15-20 min): Write clean, tested code
5. **Test** (3-5 min): Walk through test cases
6. **Optimize** (2-3 min): Discuss improvements

### Communication Tips
- **Think aloud**: Share your thought process
- **Ask questions**: Clarify ambiguities
- **Explain trade-offs**: Time vs space, simplicity vs performance
- **Handle hints**: Listen to interviewer guidance
- **Stay calm**: If stuck, take a breath and restart

### Coding Best Practices
- **Use meaningful names**: `maxProfit` not `mp`
- **Write helper functions**: Break down complex logic
- **Handle edge cases**: Empty input, single element, negatives
- **Test as you go**: Verify logic before moving on
- **Clean up**: Remove debug code, add comments

---

## Common Mistakes to Avoid
1. **Jumping to code**: Always discuss approach first
2. **Ignoring edge cases**: Test with empty, single, large inputs
3. **Poor communication**: Don't code in silence
4. **Giving up**: Keep trying, ask for hints
5. **Not testing**: Always walk through your code
6. **Messy code**: Use consistent style, proper indentation
7. **Ignoring hints**: Listen to interviewer feedback

---

## For Senior/Staff/Principal Roles

### Additional Expectations
- **System Design**: Scalability, reliability, trade-offs
- **Leadership**: Mentoring, technical direction
- **Communication**: Explain complex concepts clearly
- **Production Mindset**: Error handling, logging, monitoring
- **Trade-offs**: Discuss multiple solutions, pros/cons

### Behavioral Questions
- **STAR Method**: Situation, Task, Action, Result
- **Leadership Examples**: Mentoring, project ownership
- **Conflict Resolution**: Disagreements, compromises
- **Technical Decisions**: Architecture choices, trade-offs
- **Impact**: Quantify results (latency reduction, cost savings)

## Mock Interview Checklist
- [ ] Set up video call with timer
- [ ] Use online code editor (CoderPad, etc.)
- [ ] Practice explaining thought process
- [ ] Get feedback on communication
- [ ] Review common mistakes
- [ ] Practice system design
- [ ] Prepare behavioral stories

---

## Day Before Interview
- [ ] Review key patterns and templates
- [ ] Practice 2-3 medium problems
- [ ] Prepare questions for interviewer
- [ ] Test equipment (camera, mic, internet)
- [ ] Get good sleep
- [ ] Prepare workspace (quiet, good lighting)

---

## Day of Interview
- [ ] Arrive/login 10 minutes early
- [ ] Have water nearby
- [ ] Paper and pen for notes
- [ ] Close distractions
- [ ] Take deep breaths
- [ ] Be confident and friendly

---

## After the Interview
- [ ] Send thank-you email within 24 hours
- [ ] Note problems asked for future reference
- [ ] Reflect on what went well/poorly
- [ ] Continue practicing
- [ ] Follow up if no response in 1 week
